# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in CharClaw — the web app, the daemon, the desktop app, the agents SDK, or anything else in this repository — **please do not file a public issue**.

Instead, email **anitc98@gmail.com** with:

- A description of the vulnerability
- The version / commit you observed it on (e.g. `@charclaw/agents@0.2.0` or a git SHA)
- Steps to reproduce
- Any proposed mitigation

You will receive an acknowledgement within **5 business days**. From there, we'll work with you on a fix and coordinate disclosure timing.

## Supported Versions

Security fixes are applied to the latest minor version on `main`. Older versions are not patched unless the vulnerability is critical and the migration path is non-trivial.

| Version | Supported |
|---|---|
| 0.2.x | ✅ |
| 0.1.x | ❌ — please upgrade |

## What's in scope

- Authentication and authorization bugs (NextAuth, GitHub OAuth, workspace access)
- Data leakage (issue contents, agent output, user credentials, runtime tokens)
- Code-execution paths in the agent runtime (sandbox escape, command injection)
- Encryption issues (`ENCRYPTION_KEY` handling, credential storage)
- The daemon's local-runtime command execution surface
- The Daytona sandbox adapter and SSH transport in the SDK

## What's out of scope

- Findings on a fork or an old release that's already been superseded
- Self-XSS or attacks requiring already-compromised accounts
- Theoretical issues without a working proof of concept
- Reports auto-generated by scanning tools without analyst triage

Thanks for helping keep CharClaw and its users safe.
